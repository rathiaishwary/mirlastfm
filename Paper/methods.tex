\subsection{Feature Extraction}
\subsubsection{Albumbasierte Wirkungszeit} \label{feature_wirkungszeit}
Für die aktivste Wirkungszeit eines Künstlers wurde ein Feature (skalarer Datenwert) berechnet, das mit Hilfe der LastFM-API ermittelt wurde. Die Wirkungszeit eines Künstlers kann (muss aber nicht) repräsentativ für eine bestimmte Epoche sein. Der letzte Fall trifft vor allem bei Künstlern zu, die über mehrere Jahrzehnte hinweg aktiv sind, da sie nicht zwingend einer herkömmlichen Epoche wie den 70ern oder 80ern, etc.\ zugeordnet werden können. 

Die aktivste Wirkungszeit wurde anhand der Anzahl und des Erscheinungsjahres der veröffentlichten Alben eines Künstlers berechnet. Dazu wurden sämtliche Alben inkl. deren Releasedates mit Hilfe der LastFM-API extrahiert. Die API stellt dazu die Funktion \texttt{artist.getTopAlbums} zur Verfügung. Von den zurückgegebenen Alben können in der Folge mit der Funktion \texttt{album.getInfo} nähere Infos, u.a. das ReleaseDate des Albums, abgefragt werden.

Für die Berechnung des Features wurde von sämtlichen Releasedates ihr \textit{artithmetischer Mittelwert} gebildet. Das Ergebnis repräsentiert das mittlere Jahr der gesamten (bisherigen) Wirkungszeit. Tab.\ \ref{tab:Wirkungszeit_Features} zeigt die Ergebnisse der Featureberechnung.

\begin{equation}
 \bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i 
\end{equation}

\subsection{Ähnlichkeitsmessungen}
Die bestimmung der Ähnlichkeit zwischen 2 oder mehreren Künstlern ist eine sehr subjektive Aufgabe

\subsubsection{Differenz von skalaren Werten}
Das in Kap.\ \ref{feature_wirkungszeit} berechnete Feature wurde für die Berechnung des Ähnlichkeitsmaßes verwendet. Dieses soll darstellen, welche Künstler zur selben Zeit am aktivsten (hinsichtlich der produzierten Alben) waren. Optimalerweise lässt sich daraus auch eine gewisse Genreähnlichkeit der Künstler feststellen, da gewisse Genres, z.B. Electronic, Punk, etc. sehr typisch für gewisse Zeitepochen der Geschichte sind. Für Rock, Pop oder Heavy Metal trifft das kaum oder gar nicht zu.

Die Höhe der Ähnlichkeit wird aus der Distanz zwischen den skalaren Featurewerten zweier Künstler berechnet. Je näher sich die Features sind (ähnliche Jahreszahlen), desto geringer ist die Differenz. Folglich muss diese Distanz (nach erfolgter Normalisierung anhand der Featurewerte) invertiert werden, damit höhere Werte ein höheres Maß an Ähnlichkeit ausdrücken. Letztendlich bewegt sich diese zwischen den Werten 0.0 (keine Ähnlichkeit) und 1.0 (idente aktivste Wirkungszeit). 

\subsubsection{Cosinus-Similarity}
Ein Ma"s zur Bestimmung der "Ahnlichkeit der K"unstler in der LastFM Datenbank ist die Auswertung der Tags und ein Vergleich auf deren "Ubereinstimmung.
Dazu wurden die Tags der K"unstler mit der API ausgelesen und anschlie"send die "Ahnlichkeit mit der aus der LV bekannten Formel der Cosine Similarity bestimmt:

\begin{equation}
sim(a,b) = \frac{a*b}{\left|a\right|*\left|b\right|}
\end{equation}

Die API gibt auf Anfrage die Tags eines K"unstlers/Band mit einer Gewichtung zwischen 100 und 0 zur"uck. Zwei kleine Probleme traten hier aber auf: Zum werden auf jeden Fall eine Menge Tags zur"uckgegeben. Nach ein paar Versuchen wurde aber klar, dass die Tags mit Gewicht 0 nur zuf"alliger Natur sind und in die Berechnung nicht mit einzubeziehen sind (Mozart hat die Attribute \textit{psychodelic} bzw. \textit{Heavy-Metal} eher nicht verdient). Weiters lieferte unsere Wrapper-API f"ur Java leider nur die Tags, aber ohne Gewichtung. Die API musste daher geringf"ugig modifiziert werden.
